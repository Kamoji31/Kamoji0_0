Linked List

A Linked List is a linear data structure where each element (called a node) contains two parts: the data and a reference (or pointer) to the next node in the sequence. Unlike arrays, linked lists do not require contiguous memory allocation, which makes them dynamic in size. This structure is particularly useful for scenarios involving frequent insertions and deletions of elements, as it allows efficient modifications without shifting large amounts of data.

Advantages of Linked Lists

Linked lists offer several advantages over arrays, particularly when it comes to dynamic data management:

  • Dynamic Size: The size of a linked list can grow or shrink at runtime, unlike arrays, which have a fixed size.
  • Efficient Insertions and Deletions: Adding or removing elements can be done in constant time (O(1)) if the node to be added or removed is known, especially at the beginning or end of the list.
  • No Wasted Space: Memory is allocated as needed for each node, avoiding the wasted space associated with pre-allocated arrays.
  • Flexibility: Linked lists are versatile and can be adapted into more complex data structures such as stacks, queues, and graphs.

Common Operations on Linked Lists

Below are the most common operations that can be performed on a linked list:

  • Insertion: A new node can be added at the beginning, end, or at a specific position in the list.
  • Deletion: A node can be removed from the beginning, end, or from any specific position within the list.
  • Traversal: To access each node in the list, traversal is used. It is commonly used to print or process the data in all the nodes.
  • Search: Searching for a specific value in the list is done by traversing the list node by node until the desired value is found.
  • Reversal: Reversing a linked list involves reversing the direction of the pointers in all nodes, so the first node becomes the last, and vice versa.

Use Cases of Linked Lists

Linked lists are widely used in various applications, such as:

  • Memory Management: Operating systems use linked lists to manage memory allocation, particularly in dynamic memory allocation systems.
  • Implementation of Stacks and Queues: Linked lists are often used to implement stacks and queues due to their efficient insertions and deletions.
  • Real-time Systems: In systems where the number of elements is constantly changing, linked lists provide an efficient way to manage data.
  • Graph Representations: Linked lists can be used to represent adjacency lists in graphs, facilitating quick access and efficient space usage.

Types of Linked Lists

1. Singly Linked List

A Singly Linked List is a simple structure where each node contains data and a reference to the next node. It only allows traversal in one direction (from the head to the tail).

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

# Create nodes and link them
node1 = Node(10)
node2 = Node(20)
node1.next = node2  # Link first node to second

# Traverse the list
current = node1
while current:
    print(current.data, end=" -> ")
    current = current.next
# Output: 10 -> 20 -> None
Output
10 -> 20 -> None
2. Doubly Linked List

A Doubly Linked List is similar to a Singly Linked List, but each node also contains a reference to the previous node. This allows traversal in both directions.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None
        self.prev = None

# Create nodes and link them
node1 = Node(10)
node2 = Node(20)
node1.next = node2  # Link first node to second
node2.prev = node1  # Link second node to first

# Traverse the list forward
current = node1
while current:
    print(current.data, end=" <-> ")
    current = current.next
# Output: 10 <-> 20 <-> None
Output
10 <-> 20 <-> None
3. Circular Linked List

In a Circular Linked List, the last node points back to the first node, forming a circle. This can be either a singly or doubly linked structure.

A. Singly Circular Linked List
          class Node:
              def __init__(self, data):
                  self.data = data
                  self.next = None
          
          # Create nodes and link them in a circle
          node1 = Node(10)
          node2 = Node(20)
          node3 = Node(30)
          node1.next = node2
          node2.next = node3
          node3.next = node1  # Link last node to first node
          
          # Traverse the circular list
          current = node1
          for _ in range(4):  # Limit to 4 iterations to avoid infinite loop
              print(current.data, end=" -> ")
              current = current.next
          # Output: 10 -> 20 -> 30 -> 10 -> (circular)
Output
10 -> 20 -> 30 -> 10 -> (circular)
B. Doubly Circular Linked List
          class Node:
              def __init__(self, data):
                  self.data = data
                  self.next = None
                  self.prev = None
          
          # Create nodes and link them in a circle
          node1 = Node(10)
          node2 = Node(20)
          node3 = Node(30)
          node1.next = node2
          node2.next = node3
          node3.next = node1  # Link last node to first node
          node2.prev = node1
          node3.prev = node2
          node1.prev = node3  # Link first node to last node
          
          # Traverse the circular list forward
          current = node1
          for _ in range(4):  # Limit to 4 iterations to avoid infinite loop
              print(current.data, end=" <-> ")
              current = current.next
          # Output: 10 <-> 20 <-> 30 <-> 10 <-> (circular)
Output
10 <-> 20 <-> 30 <-> 10 <-> (circular)
