Stacks

A Stack is a linear data structure that follows the Last In, First Out (LIFO) principle. In a stack, the last element inserted is the first one to be removed. Think of a stack of plates: the plate added last will be the first one to be taken off. Stacks are widely used in scenarios like managing function calls, undo operations, and parsing expressions in programming languages.

Advantages of Stacks

Stacks offer several benefits in solving specific types of problems:

  • Efficient Operations: Both push (adding) and pop (removing) operations are executed in constant time (O(1)), making stacks efficient for scenarios with frequent additions and removals of data.
  • Memory Management: Stacks are used in many algorithms to store temporary data such as during recursive calls or function execution, reducing the complexity of memory management.
  • Simple Data Structure: The stack follows a straightforward model and is easy to implement, making it an ideal structure for managing data in various algorithms.
  • Useful for Recursion: Stacks are essential for managing the flow of recursive function calls in programs.

Common Operations on Stacks

The following operations can be performed on stacks:

  • Push: Adds an item to the top of the stack.
  • Pop: Removes the top item from the stack.
  • Peek (Top): Returns the top item without removing it from the stack.
  • isEmpty: Checks whether the stack is empty.
  • Size: Returns the number of items currently in the stack.

Real-World Use Cases of Stacks

Stacks are used in several practical applications in computer science:

  • Function Calls: Stacks are used to store the state of each function call in programming languages, enabling the program to return to the correct point after a function call finishes.
  • Undo Mechanism: In text editors or graphics software, stacks store the history of changes, allowing users to undo or redo actions in reverse order.
  • Expression Evaluation: Stacks are used in algorithms for parsing and evaluating mathematical expressions, especially for converting between infix, postfix, and prefix notation.
  • Browser History: Web browsers often use stacks to manage the history of visited pages, so users can navigate forward and backward through pages efficiently.

Stacks - Infix, Prefix, and Postfix Expressions

1. Infix Expression

An Infix expression is a common way of writing mathematical expressions, where the operator is placed between the operands. For example, in the expression `A + B`, `+` is the operator and `A` and `B` are the operands. Infix notation is easy for humans to read but can be less efficient for computation.

Example:
            # Infix Expression: A + B * C
2. Prefix Expression

A Prefix expression (also known as Polish notation) places the operator before the operands. For example, the Prefix equivalent of `A + B * C` is `+ A * B C`. This notation eliminates the need for parentheses to denote precedence of operations, making it easier for a machine to evaluate.

Example:
            # Prefix Expression: + A * B C
3. Postfix Expression

A Postfix expression (also known as Reverse Polish notation) places the operator after the operands. For example, the Postfix equivalent of `A + B * C` is `A B C * +`. This notation is also advantageous for machine evaluation since no parentheses are required and operators are applied in the order they appear.

Example:
            # Postfix Expression: A B C * +

Converting Infix to Prefix and Postfix Using Stack

Stacks play a crucial role in converting Infix expressions to Prefix or Postfix. Below are the steps for conversion:

Infix to Postfix Conversion
  1. Read the infix expression from left to right.
  2. If the character is an operand (number/variable), add it directly to the output.
  3. If the character is an operator, pop operators from the stack to the output until an operator of lower precedence or a left parenthesis is found. Then push the current operator onto the stack.
  4. If the character is a left parenthesis, push it onto the stack.
  5. If the character is a right parenthesis, pop operators from the stack to the output until a left parenthesis is encountered. Pop the left parenthesis but do not add it to the output.
  6. Once the expression has been fully read, pop all remaining operators from the stack to the output.
Example:
            # Infix to Postfix Conversion using Stack
            
            class Stack:
                def __init__(self):
                    self.stack = []
                    
                def is_empty(self):
                    return len(self.stack) == 0
                
                def peek(self):
                    return self.stack[-1]
                
                def pop(self):
                    if not self.is_empty():
                        return self.stack.pop()
                
                def push(self, item):
                    self.stack.append(item)
            
            # Operator precedence
            def precedence(op):
                if op == '+' or op == '-':
                    return 1
                if op == '*' or op == '/':
                    return 2
                return 0
            
            # Infix to Postfix Conversion
            def infix_to_postfix(expression):
                stack = Stack()
                output = []
                for char in expression:
                    if char.isalnum():  # Operand
                        output.append(char)
                    elif char == '(':  # Left Parenthesis
                        stack.push(char)
                    elif char == ')':  # Right Parenthesis
                        while stack.peek() != '(':
                            output.append(stack.pop())
                        stack.pop()  # Remove '('
                    else:  # Operator
                        while (not stack.is_empty() and precedence(stack.peek()) >= precedence(char)):
                            output.append(stack.pop())
                        stack.push(char)
                
                while not stack.is_empty():
                    output.append(stack.pop())
                    
                return ''.join(output)
            
            # Example usage
            expression = "A+B*(C-D)"
            postfix = infix_to_postfix(expression)
            print("Postfix Expression:", postfix)
Output
Postfix Expression: ABD-C*+
Infix to Prefix Conversion

To convert an Infix expression to Prefix, we follow similar steps as Infix to Postfix but with reversed operations. After obtaining the postfix expression, we can reverse it to get the prefix expression.

Prefix Evaluation Using Stack

Once you have a Prefix expression, you can evaluate it using a stack by following these steps:

  1. Read the prefix expression from right to left.
  2. If the character is an operand, push it onto the stack.
  3. If the character is an operator, pop two operands from the stack, apply the operator, and push the result back onto the stack.
  4. After reading the entire expression, the final result will be the single value remaining in the stack.
Postfix Evaluation Using Stack

To evaluate a Postfix expression, you can follow the steps below:

  1. Read the postfix expression from left to right.
  2. If the character is an operand, push it onto the stack.
  3. If the character is an operator, pop two operands from the stack, apply the operator to the operands, and push the result back onto the stack.
  4. Continue this process until all characters of the postfix expression have been processed.
  5. At the end of the evaluation, the stack will contain a single element, which is the result of the postfix expression.
Example:
            # Postfix Evaluation using Stack
            
            class Stack:
                def __init__(self):
                    self.stack = []
                
                def is_empty(self):
                    return len(self.stack) == 0
                
                def pop(self):
                    if not self.is_empty():
                        return self.stack.pop()
                
                def push(self, item):
                    self.stack.append(item)
            
            def evaluate_postfix(expression):
                stack = Stack()
                
                for char in expression:
                    if char.isdigit():  # If the character is a digit (operand)
                        stack.push(int(char))
                    else:  # If the character is an operator
                        operand2 = stack.pop()
                        operand1 = stack.pop()
                        
                        if char == '+':
                            result = operand1 + operand2
                        elif char == '-':
                            result = operand1 - operand2
                        elif char == '*':
                            result = operand1 * operand2
                        elif char == '/':
                            result = operand1 / operand2
                        
                        stack.push(result)
                
                return stack.pop()  # The final result will be the only element left in the stack
            
            # Example usage
            expression = "23*5+"
            result = evaluate_postfix(expression)
            print("Postfix Evaluation Result:", result)
Output
Postfix Evaluation Result: 11

In the example above, the expression "23*5+" is evaluated as follows:

  • First, 2 and 3 are pushed onto the stack.
  • The '*' operator is encountered, so 2 and 3 are popped from the stack, multiplied (2 * 3 = 6), and the result (6) is pushed back onto the stack.
  • Next, 5 is pushed onto the stack.
  • The '+' operator is encountered, so 6 and 5 are popped from the stack, added (6 + 5 = 11), and the result (11) is pushed back onto the stack.
  • At the end, the stack contains the result 11, which is the final result of the postfix expression.
